## 实验设置

### 探测器系统
- **上探测器**: 闪烁体 + PMT
- **下探测器**: 闪烁体 + PMT
- **探测器间距**: 16.2 cm（部分实验为50 cm）
- **探测器尺寸**: 约10×10 cm²

### 数据采集
- **数字化仪**: CAEN DT5742
- **采样率**: 5 GHz (5 GS/s)
- **采样间隔**: 0.2 ns
- **波形长度**: 1024个采样点 (204.8 ns)
- **触发模式**: 符合触发或单探测器触发

### 铅板配置
- **材料**: 纯铅（纯度>99%）
- **单片厚度**: 0.5 mm
- **放置位置**: 两探测器之间
- **对准精度**: ±1 mm

## 数据内容

每个ROOT文件包含一个Tree (`t1`)，记录每个事件的测量值：

### 存储变量
- `area`: 上探测器脉冲积分（能量相关）
- `area2`: 下探测器脉冲积分（能量相关）
- `Index_minamp1`: 上探测器波形最小值位置（时间信息）
- `Index_minamp2`: 下探测器波形最小值位置（时间信息）
- `minamp1`: 上探测器最小幅度
- `minamp2`: 下探测器最小幅度

### 典型事件数
- 每个文件: 约50,000 - 200,000个事件
- 测量时间: 1小时
- 触发率: 约14-56 Hz

## 使用方法

### 读取数据
```python
import uproot
import numpy as np

# 打开文件
with uproot.open('5181.root') as f:
    tree = f['t1']
    
    # 读取数据
    area = tree['area'].array(library='np')
    area2 = tree['area2'].array(library='np')
    index1 = tree['Index_minamp1'].array(library='np')
    index2 = tree['Index_minamp2'].array(library='np')
```

### 符合事件选择
```python
# 定义阈值
area_threshold = 30
area2_threshold = 30

# 符合条件
coincidence = (area > area_threshold) & (area2 > area2_threshold)

# 提取符合事件
area_coin = area[coincidence]
area2_coin = area2[coincidence]
```

### 时间差计算
```python
# 采样率5 GHz，采样间隔0.2 ns
sampling_interval_ns = 0.2

# 时间差（纳秒）
time_diff_ns = (index2 - index1) * sampling_interval_ns
```

### 数据完整性

- 验证Tree结构: `tree->Print()`

### 系统效应
- 不同周次测量可能有系统偏移
- 温度变化影响PMT增益
- 高压稳定性影响阈值
- 探测器位置可能微调

## 数据分析建议

### 基准测量
始终使用5181.root（0片铅板）作为基准：
- 归一化其他测量
- 评估系统漂移
- 计算相对变化

### 分组分析
- **上周数据** (5181-5184): 内部一致，可直接比较
- **本周数据** (5185-5186): 需要归一化处理
- **跨周比较**: 使用Coin/Up ratio等归一化方法

### 统计误差
- 泊松统计: σ = √N
- 1小时测量: N~3000, σ~55 (1.8%)
- 相对误差小，系统误差占主导

## 物理目标

### 主要测量
1. **宇宙射线速度**: 使用时间差和探测器间距
2. **Rossi转换曲线**: 符合率vs铅厚度
3. **衰减特性**: 粒子在铅中的衰减
4. **能量分布**: area和area2谱

### 预期结果
- 速度: ~0.99c
- Rossi峰: 理论上在2-3 X₀（实际未观察到）
- 衰减: 主要由多重散射导致
- 能量: 典型宇宙射线μ子能谱

## 常见问题

**Q: 为什么文件大小不同？**
A: 触发率不同导致事件数不同，铅板越厚，符合率越低。

**Q: 如何获取原始波形？**
A: 需要修改数据采集程序保存完整波形，当前只保存提取的参数。

**Q: 数据可以共享吗？**
A: 请遵守实验室数据管理政策，引用时注明来源。

**Q: 如何处理损坏的文件？**
A: 尝试用ROOT的恢复工具，或重新测量。

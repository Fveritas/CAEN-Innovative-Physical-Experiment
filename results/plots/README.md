**内容**: 宇宙射线速度分布分析（双面板）

**左图 - 时间差分布**:

**物理意义**:
- 验证宇宙射线μ子的相对论性质
- 速度接近光速说明粒子能量高（>几百MeV）
- 分布宽度反映能量谱和测量误差

### 2. Rossi曲线 (`rossi/`)

#### `rossi_curve.png`
**内容**: 基础Rossi转换曲线（v1）

**图表元素**:
- 横轴: 铅板厚度（mm或X₀）
- 纵轴: 符合计数率（counts/h）
- 数据点: 不同厚度的测量值
- 误差棒: 统计误差（√N）
- 拟合曲线: 可能的趋势线

**预期vs实际**:
- 预期: 先上升（簇射发展）→峰值（Rossi峰）→下降（吸收）
- 实际: 单调下降（无明显Rossi峰）

### 3. 衰减分析 (`attenuation/`)

#### `lead_attenuation.png`
**内容**: 铅衰减曲线

**可能包含**:
- **线性坐标图**: 直观显示衰减
  - 横轴: 铅厚度（mm或g/cm²）
  - 纵轴: 符合率（counts/h）
  - 拟合: 指数衰减 N = N₀×exp(-μx)
  
- **半对数图**: 检验指数规律
  - 纵轴: log(N)
  - 应为直线（斜率=-μ）
  
- **归一化图**: 相对于N₀
  - 纵轴: N/N₀
  - 清晰显示衰减比例

**拟合参数**:
- N₀: 初始计数率
- μ: 衰减系数（cm⁻¹）
- x₁/₂: 半值层厚度
- χ²/ndf: 拟合质量

**物理解释**:
- 衰减主要由多重散射导致
- 而非能量损失（μ子穿透力强）
- 几何效应占主导

#### `diagnostic_v4_2d.png` ⭐
**内容**: 2D诊断图（最新版）

**主图 - area vs area2散点图**:
- 横轴: area（上探测器）
- 纵轴: area2（下探测器）
- 颜色: 事件密度（热图）
- 阈值线: area=30, area2=30
**区域划分**:
1. **左下角**（双低）:
   - area < 30, area2 < 30
   - 噪声事件
   - 应该被阈值过滤

2. **右下角**（上高下低）:
   - area > 30, area2 < 30
   - 单上探测器触发
   - 粒子未到达下探测器或能量不足

3. **左上角**（上低下高）:
   - area < 30, area2 > 30
   - 单下探测器触发
   - 罕见（宇宙射线从上往下）

4. **右上角**（双高）:
   - area > 30, area2 > 30
   - **符合事件**（目标信号）
   - 穿过两个探测器的粒子

**应用**:
- 优化阈值设置
- 评估符合纯度
- 识别噪声来源
- 检测探测器问题

**理想情况**:
- 大部分事件在右上角
- 左下角噪声少
- 对角线分布（能量相关）

## 图表使用指南

### 查看图表
```bash
# Linux
xdg-open velocity/5181b_velocity.png

# 或使用图像查看器
eog velocity/5181b_velocity.png
```

### 在报告中使用
- 所有图表为PNG格式，适合插入文档
- 分辨率: 150 DPI（打印质量）
- 尺寸: 通常1200-1800像素宽

### 重新生成
运行相应的分析脚本会覆盖现有图表：
```bash
cd ../../analysis/cosmic_rays
python calculate_velocity.py ../../data/raw/5181.root 50.0
```

## 图表解读技巧

### 1. 检查坐标轴
- 单位是否正确
- 刻度是否合理
- 对数还是线性

### 2. 评估误差棒
- 是否显示
- 大小是否合理
- 统计还是系统误差

### 3. 观察趋势
- 是否符合物理预期
- 有无异常点
- 系统性偏离

### 4. 比较不同图表
- 一致性检查
- 互相验证
- 发现新规律

## 常见问题

**Q: 图表模糊怎么办？**
A: 修改脚本中的dpi参数（如dpi=300）重新生成。

**Q: 如何修改图表样式？**
A: 编辑相应的Python脚本，修改matplotlib参数。

**Q: 可以导出为其他格式吗？**
A: 可以，修改savefig的文件扩展名（.pdf, .svg, .eps）。

**Q: 图表太多，如何管理？**
A: 按分析类型分类存放，使用描述性文件名，定期清理旧版本。
